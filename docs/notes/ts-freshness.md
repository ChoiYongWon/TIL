# 신선도

> 타입스크립트는 객체의 "신선도(Freshness)"에 따라 타입 검사의 엄격성을 다르게 적용한다

- 높은 신선도 (객체 리터럴 직접 전달)

  - 객체 리터럴을 직접 함수에 전달할 때 엄격한 검사

- 낮은 신선도 (변수 할당 후 전달)

  - 변수에 할당 후 전달할 때 재할당 가능성을 고려하여 타입을 더 넓게 해석

신선도는 객체가 생성된 직후의 상태를 의미한다. 객체 리터럴이 직접 사용될 때는 "신선한" 상태이지만, 변수에 할당되면 그 신선도가 사라진다.
